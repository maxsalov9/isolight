# IsoLight — Minimal Art Direction v0.1

**Игра:** IsoLight  
**Документ:** Minimal Art Direction / Visual Prototype Pass  
**Версия:** 0.1  
**Цель:** улучшить визуальное восприятие первого прототипа, не уходя в финальный арт и не тормозя разработку механик.

---

# 1. Главная цель визуального прохода

Сейчас задача не в том, чтобы сделать красивую финальную графику.

Задача:

> Сделать так, чтобы даже на placeholder-уровне игрок понял атмосферу IsoLight: темный разрушенный район, редкие островки света, важные системы поселения и конфликт вокруг энергии.

Минимальная графика должна помогать:

- читать локацию;
- понимать, где важные объекты;
- отличать персонажей;
- чувствовать постапокалиптический мир;
- видеть тему “электричество = жизнь”.

---

# 2. Главный принцип

```text
Не финальный арт.
Не красивые ассеты ради красоты.
Только читаемость, атмосфера и игровая функция.
```

Правильный порядок:

```text
gray boxes → readable zones → lights → props → mood → polish
```

Неправильный порядок:

```text
final models → custom textures → complex VFX → а потом когда-нибудь gameplay
```

---

# 3. Визуальная формула IsoLight

## 3.1. Основная формула

**Холодные мокрые руины + теплые островки электрического света + технологические объекты выживания.**

Игрок должен сразу видеть:

- мир темный и сломанный;
- но отдельные зоны еще живые;
- свет редкий и важный;
- каждая включенная лампа означает ресурс, людей, власть.

---

## 3.2. Цветовой принцип

| Элемент | Цвет / ощущение |
|---|---|
| Руины | холодный серый, темно-синий, грязно-зеленый |
| Асфальт | темный, влажный, почти черный |
| Бетон | серый, холодный, грязный |
| Растительность | приглушенный зеленый, не яркий |
| Рабочий свет | теплый желтый / янтарный |
| Электричество / tech | синий / голубой |
| Опасность | красный / грязно-оранжевый |
| Важные интерактивные объекты | теплый контур или мягкая подсветка |
| UI | темные панели + янтарные/синие акценты |

---

# 4. Riverside District — минимальное визуальное настроение

## 4.1. Что должно ощущаться

Riverside — это не пустая карта.  
Это поселение, которое еще держится, но уже трещит.

Визуально:

- мокрый асфальт;
- старые здания;
- кабели между стенами;
- тусклые лампы;
- контейнеры воды;
- предупреждения;
- генератор;
- ферма;
- фильтры;
- ворота;
- сцена;
- мастерская;
- релейная антенна;
- люди, спорящие за свет.

---

## 4.2. Время суток

**Сумерки / поздний вечер.**

Почему:

- свет становится важнее;
- тени делают мир опаснее;
- теплые лампы лучше контрастируют с холодным окружением;
- тема “островков света” читается сильнее.

# 6. Минимальные props для читаемости

Не нужно делать много пропсов. Нужны только те, которые объясняют функцию зоны.

## 6.1. Filter Station

Чтобы игрок понял “это вода / фильтрация”:

- 2–3 цилиндрических бака;
- трубы;
- канистры;
- табличка `DO NOT DRINK OPEN WATER`;
- маленький терминал качества воды;
- холодная синяя подсветка;
- несколько ведер/емкостей.

## 6.2. Hydroponic Farm

Чтобы игрок понял “это еда / растения / будущее”:

- стеллажи;
- зеленые панели/плашки растений;
- grow lights;
- насос;
- трубы;
- резервуар питательного раствора;
- теплый свет внутри;
- несколько “мертвых” лотков.

## 6.3. Defense Gate

Чтобы игрок понял “это оборона”:

- ворота;
- забор;
- проволока;
- прожекторы;
- баррикады;
- мешки/блоки;
- красные предупреждающие элементы.

## 6.4. Public Stage

Чтобы игрок понял “это сцена / собрания / мораль”:

- небольшая платформа;
- старый прожектор;
- микрофон;
- 2–3 лавки/стула;
- афиша/доска объявлений;
- теплый слабый свет.

## 6.5. Workshop

Чтобы игрок понял “это ремонт”:

- верстак;
- ящики;
- инструменты;
- кабели;
- зарядная стойка;
- искрящийся щиток;
- ржаво-оранжевая подсветка.

## 6.6. Relay Station

Чтобы игрок понял “это связь / старая сеть”:

- антенна;
- терминал;
- кабельный шкаф;
- мигающая синяя лампа;
- старый знак сети;
- динамик/радиоблок.

## 6.7. Generator Yard

Чтобы игрок понял “это центр миссии”:

- крупный генератор G-17;
- кабели;
- щиток;
- аккумуляторные блоки;
- укрытия;
- прожектор;
- warning light;
- место для боя.

# 7. Layout readability

## 7.1. Правило зон

Каждая зона должна читаться по силуэту и цвету.

| Зона | Визуальный маркер |
|---|---|
| Filter Station | баки + трубы + синий/белый свет |
| Hydroponic Farm | зеленые стеллажи + grow lights |
| Defense Gate | ворота + прожекторы + забор |
| Public Stage | платформа + микрофон + теплый свет |
| Workshop | верстак + инструменты + ржавый свет |
| Relay Station | антенна + синий терминал |
| Generator Yard | большой генератор + кабели |

---

## 7.2. Не перегружать

На первом visual pass не нужно ставить:

- десятки мусорных объектов;
- много разных зданий;
- сложные интерьеры;
- реалистичный город;
- тяжелую растительность;
- финальные модели NPC.

Лучше меньше объектов, но каждый объясняет функцию.

---

# 8. Lighting direction

## 8.1. Главный свет

- холодный общий ambient;
- мягкая темно-синяя атмосфера;
- мало общего света;
- больше локальных источников.

## 8.2. Локальные источники

| Источник | Цвет |
|---|---|
| обычные лампы поселения | теплый желтый |
| электрические системы | голубой |
| аварийные лампы | красно-оранжевый |
| ферма | теплый grow light |
| фильтры | холодный бело-синий |
| сцена | теплый янтарный |
| реле | синий мигающий |

---

## 8.3. Цель света

Свет должен не просто украшать сцену.  
Он должен показывать:

- куда идти;
- что важно;
- что работает;
- что выключено;
- где жизнь;
- где опасность.

---

# 9. Персонажи на placeholder-этапе

## 9.1. Не нужны финальные модели

Пока достаточно:

- capsule / low-poly humanoid;
- разный цвет акцента;
- разный размер;
- маленький prop рядом или на спине.

## 9.2. Силуэты

| Персонаж | Placeholder silhouette |
|---|---|
| Dax | чуть шире, рюкзак/ящик за спиной |
| Nyra | тоньше, маленький “tablet” prop |
| Cormac | средний, medical pack / cross marker |
| Hale | высокий/темный, рядом забор/оружейный ящик |
| Sela | светлый/медицинский, рядом бак/стол |
| Ivo | зеленый акцент, рядом стеллаж |
| Edda | выделяющийся цвет, рядом микрофон |
| Greer | коренастый, рядом верстак |
| Lysa | синий акцент, рядом терминал |

# 10. UI visual pass

Пока UI может оставаться placeholder, но его нужно сделать читаемым.

## 10.1. Минимальный стиль UI

- темный полупрозрачный фон;
- белый текст;
- янтарный accent для active objective;
- голубой accent для tech/energy;
- красный accent для warning;
- без декоративного шума.

## 10.2. Что улучшить

- Objective Panel;
- Interaction Prompt;
- Notification UI;
- Character selection indicator;
- NPC name labels;
- Quest markers.

---

# 11. Что можно сделать через Codex

Codex может сделать минимальный визуальный pass без финального арта:

- создать материалы;
- назначить цвета placeholder-объектам;
- добавить примитивные props;
- обновить scene builder;
- добавить простые lights;
- добавить zone labels;
- добавить простые icon markers;
- сделать visual result states позже.

---

# 12. Что НЕ просить у Codex сейчас

Пока не просить:

- создать финальных персонажей;
- сделать красивые текстуры;
- добавить сложные модели;
- импортировать ассеты из интернета;
- подключать платные пакеты;
- делать сложный post-processing;
- делать cinematics;
- оптимизировать графику.

# 15. Рекомендованный порядок

Лучше сделать так:

1. Batch 5 — Dialogue.
2. Batch 5.5 — Minimal Visual Prototype Pass.
3. Batch 6 — Power Politics.
4. Batch 7 — Combat.

Почему:

- диалоги сделают поселение живым;
- visual pass сделает поселение читаемым;
- power politics даст выбор;
- combat даст напряжение.

---

# 16. Критерии успеха visual pass

Сцена успешна, если игрок за 10 секунд понимает:

- где фильтры;
- где ферма;
- где ворота;
- где сцена;
- где мастерская;
- где реле;
- где генератор;
- кто из персонажей Dax, Nyra, Cormac;
- что мир темный, но отдельные точки живут за счет света.

---

# 17. Краткое резюме

Minimal Art Direction нужен не для красоты.

Он нужен, чтобы IsoLight уже на раннем прототипе ощущался как игра про:

- темноту;
- дефицит;
- инфраструктуру;
- электричество;
- людей, спорящих за свет.

Главное правило:

> Свет должен быть не декором.  
> Свет должен быть смыслом.
